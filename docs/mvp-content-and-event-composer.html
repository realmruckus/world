MVP 内容规模与 Event Composer

每次行动应提供可预期基础收益、小概率高价值惊喜、可积累保底进度，以及可能改变角色或世界的事件。

MVP 内容量

对象建议规模
核心事件模板80–120
玩家可见结果变体300–600
视觉资产100–160，其中大量为图标
卡牌式数据对象90–140
基础可选种族3
可招募或遭遇怪物6–10
具名地点15–25

Event Composer 输入

  • 行动、地点、区域等级、环境与 World Tick。
  • 怪物种族、属性、技能、装备、伤势、特性、经历和关系。
  • 近期事件历史、稀有保底进度、地点资源与控制状态。
  • 可见的其他玩家异步事件。

输出

  • 标题、情境文本、可选行动和条件。
  • 结构化结果分支、奖励、损失和长期 Effect。
  • 角色、地点和世界变化。
  • 后续事件钩子、冷却和重复权重。

必要约束

  • 文本生成与规则结算分离。
  • 只能使用批准的数据池，不得自行创造物品、怪物或数值。
  • 模板版本、随机种子和结果可重放、审计。
  • 短时间避免重复；高价值奖励受地点、难度和保底约束。

资产策略

MVP 不为每个事件制作独立插画。优先复用怪物立绘、地点背景、装备图标、状态图标和卡牌版式。成长阶段可通过装备叠层、徽记和边框变化表达。

时间估算

约 3 个月仅适用于严格受控的快速验证版；包含稳定事件内容、账号存档、异步投影、Simulation 和公开测试准备的 MVP，更稳妥按 4–6 个月评估。该范围不是正式排期。
