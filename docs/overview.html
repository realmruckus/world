Design reference

WORLD 模式总览

WORLD 是以卡牌和事件选择作为交互语言的异步共享世界怪物角色扮演与区域争夺游戏。

核心循环

选择怪物与地点
→ 选择狩猎、采集、探索、侦查、进攻等行动
→ Event Composer 选择结构化事件
→ 玩家作出选择
→ Effect 修改怪物卡、资源、关系与世界
→ World Tick 推进
→ 看到其他玩家在相同进度留下的痕迹
→ 驻守、合作或抢夺地点

长期状态

  • 账号共享：World Tick、金币、材料、仓库、领地、怪物栏位、图鉴与世界事件记录。
  • 怪物独立：种族、等级、属性、技能、特性、经历、关系、装备、伤势、位置与状态。

卡牌职责

类型职责
行动狩猎、探索、采集、侦查、驻守、进攻、撤退等选择。
技能由种族、经历与成长解锁，改变事件条件和结果。
装备提供属性、行动修正、特殊选项或地点适应能力。
道具一次性改变事件、风险或奖励。
经历记录重大事件并影响后续文本与规则。

多人世界

WORLD 不要求实时位置、实时战斗或多人同屏。玩家通过追加式世界事件相互影响；地点控制、设施、防守快照和异步世界痕迹在查看者达到对应 Tick 后可见。

经济与成长

金币、材料、装备、驻军维护、地点产出和进攻损失形成循环。所有数值必须通过 Simulation 验证通胀、滚雪球、地点价值和新玩家追赶能力。

边界

本页描述 WORLD 产品方向。通用 Mechanic、Effect 约定、产品元数据和长期项目基线仍由 Core 维护；WORLD 的应用、产品专属规则、内容、网站、引擎适配和版本迭代由本仓库维护。
