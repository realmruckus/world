M00 · Life Simulation Exploration

现实人生重开探索验证程序开发计划

本程序不是 WORLD 正式玩法，而是用于验证事件驱动人生模拟的前置实验。它必须独立、单页、单机、无数据库，重点验证核心循环、事件组织、分支反馈、档案记录、多周目驱动力和内容扩充方式。

一、验证目标

  • 核心循环

    出生 → 年龄推进 → 事件选择 → 状态变化 → 关系与经历累积 → 结局 → 成就 → 再开一生。

  • 事件可玩性

    验证现实事件是否具有辨识度、选择压力、意外性、长期后果和重复游玩价值。

  • 内容结构

    验证 JSON 是否足以表达条件、权重、分支、链式事件、延迟后果、冷却、稀有度与结局。

  • 档案价值

    验证人生时间线、人物关系、重大经历、统计摘要、死因和成就是否能形成可分享的个人故事。

  • WORLD 复用

    验证 Event Composer、状态条件、Effect、事件日志、角色档案和多轮历史能否迁移到 WORLD 的怪物与世界事件系统。

二、产品边界

本阶段包含

  • 现实世界背景
  • 单人物完整生命周期
  • 家庭、教育、工作、关系、财富、健康和社会事件
  • 多分支事件与长期状态
  • 多样结局、人生评分、跨周目成就
  • JSON 内容、本地存储、JSON 导入导出
  • 程序生成 SVG 人物和场景示意图

本阶段不包含

  • 账号系统和远程数据库
  • 在线多人、排行榜和云存档
  • 真实货币、广告和商业化
  • 复杂地图或自由移动
  • 完整社会经济模拟
  • 生成式 AI 在线调用
  • 正式 WORLD 怪物、领地和战斗规则

三、核心游戏循环

创建角色
  ↓
生成出生背景、家庭和初始资质
  ↓
按年龄推进 1 年
  ↓
筛选可触发事件
  ↓
按优先级、权重、冷却和保底抽取事件
  ↓
玩家选择分支
  ↓
执行即时 Effect + 写入延迟后果 + 更新关系/经历/标签
  ↓
记录人生档案并检查链式事件、重大节点和结局
  ↓
继续下一年 / 结束人生
  ↓
生成结局摘要、评分、成就与历史人生档案
  ↓
重新投胎

单次完整人生目标时长为 8–15 分钟。每年原则上只展示一个主要事件，必要时附带一个自动结算的小事件，避免信息密度失控。

四、人物状态模型

基础身份

姓名、性别表达、出生年份、地区、家庭结构、成长环境、教育阶段、职业、婚姻状态、子女、居住状态。

核心属性

健康、心情、智力、魅力、体能、纪律、同理心、压力。统一使用 0–100，并由事件 Effect 调整。

资源

现金、收入、债务、资产、职业等级、教育资格、社会声望、空闲时间。

关系

父母、兄弟姐妹、伴侣、子女、朋友、同事。每段关系保存亲密度、信任、冲突、依赖、状态和共同经历。

标签与经历

性格标签、健康标签、教育标签、职业标签、家庭标签、法律标签、重大人生经历和隐藏倾向。

长期变量

累计幸福、累计压力、事业稳定度、关系稳定度、财务安全度、风险倾向、遗憾和人生转折次数。

五、事件类型

事件按领域、生命周期、触发方式和叙事功能四个维度分类。单个事件可以同时属于多个标签。

1. 出生与家庭

家庭收入、父母关系、兄弟姐妹、搬家、家庭冲突、家庭支持、照护责任、继承与失去亲人。

2. 童年成长

兴趣启蒙、校园适应、朋友、欺负与被欺负、比赛、意外、家庭期待、性格形成。

3. 教育

升学、选科、考试、奖学金、退学、复读、实习、留学、技能训练和教育债务。

4. 职业

求职、入职、绩效、升职、降职、裁员、转行、创业、职业倦怠、职场关系和退休。

5. 财富与消费

工资、存款、债务、买房、租房、投资、保险、意外开销、诈骗、破产、继承和退休金。

6. 友谊与社交

结识朋友、友情考验、误会、合作、背叛、重逢、社交圈变化和孤独。

7. 恋爱与婚姻

相识、约会、表白、分手、复合、同居、求婚、婚姻冲突、离婚、再婚和伴侣照护。

8. 生育与家庭责任

是否要孩子、育儿压力、教育选择、代际冲突、子女成长、空巢和家庭照护。

9. 健康与生活方式

疾病、检查、治疗、运动、饮食、睡眠、心理压力、慢性病、康复和老年照护。

10. 法律与风险

交通事故、纠纷、违法诱惑、被骗、诉讼、处罚、犯罪记录和社会后果。只做抽象化、非教学性的结果表达。

11. 公共与时代事件

经济衰退、行业变化、公共卫生、技术替代、自然灾害、政策变化和社会潮流。使用虚构年份或抽象现实背景，避免依赖实时新闻。

12. 机会与奇遇

贵人、偶然机会、意外发现、中奖、小概率逆转、突发邀请和稀有人生转折。

13. 价值观与抉择

诚信与利益、家庭与事业、稳定与冒险、个人愿望与责任、宽恕与追责。

14. 晚年与回顾

退休安排、健康衰退、伴侣与朋友离世、代际关系、遗产安排、人生回顾和最后愿望。

六、事件触发模式

  • 普通随机事件：满足年龄和状态后进入权重池。
  • 里程碑事件：入学、成年、毕业、退休等关键年龄或状态必触发。
  • 链式事件：由前置事件写入 flag，后续在指定年龄或条件下继续。
  • 延迟后果：当前选择写入 futureEffects，在若干年后自动结算或触发新事件。
  • 关系事件：由某段关系的亲密度、冲突值和状态触发。
  • 资源事件：由现金、债务、职业等级、住房和教育资格触发。
  • 危机事件：低健康、高压力、破产、失业或关系崩溃时进入保底池。
  • 稀有事件：低基础权重，并可受幸运、声望、经历和成就修正。
  • 结局前事件：接近死亡或满足重大结局条件时触发最后回顾与选择。

事件模板只表达内容和规则数据。筛选、随机、Effect 执行、日志和存档由 Engine 负责，页面不得直接写死结算逻辑。

八、分支与后果层级

即时反馈

属性、金钱、关系和标签立即变化，并在选择后明确显示。

年度后果

下一年或若干年后结算收入、债务、健康和关系变化。

链式后果

开启专属事件线，如创业、婚姻、疾病康复、犯罪记录或家庭照护。

结构性后果

改变后续事件池、可选职业、关系对象、收入上限、健康风险和结局资格。

叙事回收

早期选择应在中晚年以回忆、关系反应、遗憾、荣誉或结局文本回收。

隐藏检定

部分选择基于属性、标签和随机种子判定不同结果，但必须通过结果说明让玩家理解大致原因。

九、结局体系

结局由“生命结束原因 + 人生主结局 + 次级标签 + 统计摘要”组成，不只输出一个固定标题。

生命结束原因

  • 自然寿终
  • 疾病
  • 意外
  • 公共灾害
  • 高风险行为的抽象结果
  • 特殊剧情结局

人生主结局类别

平凡而完整

没有极端成就，但家庭、健康和生活相对稳定。

事业巅峰

职业等级、声望、收入与重大职业经历达到高位。

创业传奇

创业链完成，并在财富、影响力或持续经营方面达标。

财富自由

净资产与被动收入足以覆盖晚年生活，债务风险低。

桃李满门

教育、指导、子女或后辈关系形成强烈正向影响。

家庭支柱

承担长期照护与责任，家庭关系总体稳固。

挚爱相伴

长期伴侣关系稳定，并共同经历多个关键事件。

自由灵魂

未以传统事业或家庭为主，但探索、兴趣和自主性极高。

学术与创造

教育、研究、艺术、写作或技术创造经历达到高位。

公共贡献

社会服务、公益、医疗、教育或公共职业影响显著。

大起大落

经历多次财富、事业、关系或身份剧烈反转。

孤独晚年

关系网络严重萎缩，亲密度和社会支持不足。

债务人生

长期债务和财务危机主导人生，且未能完成恢复链。

未竟之路

重要目标或链式事件未完成，遗憾值较高。

重新出发

中晚年完成重大转行、康复、再婚或财务重建。

稀有隐藏结局

由特定事件组合、极端属性或跨周目成就解锁。

次级结局标签

健康长寿、富有、负债、受人尊敬、家庭和睦、子女疏远、朋友众多、孤独、冒险家、守序者、终身学习、连续创业、频繁转行、晚年重生等。主结局和次级标签共同构成人生摘要。

十、人生评分

总分不作为唯一胜负条件，只用于比较不同人生。建议显示六项分数和一项综合评价：

  • 健康与寿命
  • 幸福与心理稳定
  • 关系与家庭
  • 事业与创造
  • 财富与安全
  • 经历丰富度
  • 综合人生分：加权结果，保留负面经历带来的故事价值，不简单等同于成功。

十一、跨周目成就

基础成就

完成第一生、活到 80 岁、第一次结婚、第一次买房、第一次退休。

职业成就

体验 10 种职业、成为行业顶尖、完成一次成功转行、创业成功。

关系成就

维持长期友谊、三代同堂、终身伴侣、与家庭和解。

挑战成就

从贫困背景实现财务安全、从重大疾病康复、破产后重建、低属性开局完成高评价。

收集成就

触发全部事件类型、发现稀有事件、获得不同主结局、解锁隐藏事件链。

多人生元成就

累计完成 5、10、25 次人生；连续三生走完全不同路线；获得全部主结局类别。

成就只解锁事件提示、统计与轻量初始选项，不在验证阶段引入强数值继承，避免多周目变成单向碾压。

十二、档案与本地数据

  • 当前人生：完整状态、随机种子、年龄、事件队列、关系、标签、经历、日志和版本。
  • 历史人生：角色摘要、出生与死亡、主结局、次级标签、总分、重大节点和关键统计。
  • 全局档案：完成次数、事件发现率、结局收藏、成就、最佳纪录和累计统计。
  • 设置：文字速度、减少动画、调试信息和数据版本。
  • 导出：单个 JSON 文件包含 schemaVersion、当前人生、历史人生、成就和设置。
  • 迁移：每次结构变化必须提供本地存档迁移函数，不允许静默丢弃旧数据。

十三、程序化 SVG 视觉

  • 人物头像由脸型、发型、发色、肤色、服饰、年龄线条和表情组合生成。
  • 事件场景使用办公室、学校、住宅、医院、法庭、街道、机场等抽象 SVG 图标与背景。
  • 视觉只负责辅助辨识事件类型，不承担核心信息，所有状态必须有文字说明。
  • SVG 参数由人物状态和事件 visual 字段驱动，不保存独立图片文件。
  • 禁止依赖第三方图片、远程字体或在线图像服务。

十四、内容规模

Playable Prototype

40–60 个事件模板、8 个主结局、12 个成就、4 类职业、基础家庭和关系。

探索验证版

120–180 个事件模板、16 个主结局、30–40 个成就、12 类职业、20 条短事件链。

重复度目标

连续三次人生中，相同主要事件不超过 35%；每次人生至少出现 2 个不同于上一轮的重大节点。

事件分布

普通 65%、条件事件 20%、链式事件 10%、稀有事件 5%。实际权重需通过批量 Simulation 校准。

十五、开发阶段

  1. P0 数据与规则冻结：确定状态字段、Effect 操作、事件 JSON Schema、结局计算和存档版本。
  2. P1 Engine 垂直切片：角色生成、年龄推进、事件筛选、权重抽取、选择结算、日志和基础死亡。
  3. P2 页面交互：创建角色、事件选择、属性反馈、时间线、历史人生、成就和 SVG。
  4. P3 内容扩充：完成 40–60 个首轮事件，并覆盖所有生命周期和主要事件类型。
  5. P4 链式事件与结局：加入延迟后果、关系链、职业链、健康链和至少 8 个主结局。
  6. P5 本地持久化：自动存档、恢复、历史档案、跨周目成就、导入导出和迁移。
  7. P6 调试与 Simulation：固定种子重放、批量人生模拟、事件覆盖率、死亡年龄、结局分布和重复度统计。
  8. P7 移动端审核：在 iPhone 竖屏完成完整人生，检查按钮、文字长度、滚动、存档和部署链接。

十六、自动测试

  • 相同种子、相同输入得到相同事件和结果。
  • 年龄、标签、属性、资源和关系条件筛选正确。
  • 冷却、oncePerLife、链式事件和延迟后果不重复或丢失。
  • Effect 操作只能修改允许字段，非法路径必须拒绝。
  • 所有数值属性保持合法范围，金钱和债务规则一致。
  • 死亡后不能继续推进，结局只能结算一次。
  • 导出后再导入得到等价状态。
  • 旧 schemaVersion 可迁移到当前版本。
  • 批量模拟不存在无事件可选、永不结束或明显单一结局。
  • 全部事件引用的 ID、职业、标签、场景和后续事件均有效。

十七、好玩性验证指标

  • 第一次游玩能否在 30 秒内理解“推进年龄—做选择—看后果”。
  • 选择是否至少有两项看起来合理，而不是明显正确答案。
  • 关键选择是否在 3–10 年内产生可识别回报或代价。
  • 一生中是否至少形成 3 个可复述的故事节点。
  • 结局是否能解释玩家为什么得到该评价。
  • 结束后是否有明确动力再开一生：未见事件、未获结局、成就或尝试相反路线。
  • 三轮游玩后事件重复和文本疲劳是否可接受。

十八、完成标准

  • 正式页面部署到 https://world.realmruckus.com/life-sim.html 并可在移动端访问。
  • Engine、事件 JSON、页面、SVG 生成和存档模块职责分离。
  • 至少 40 个事件、8 个主结局、12 个成就可运行。
  • 完整人生、死亡、结局、历史档案和重新投胎流程无阻塞。
  • 自动测试、语法检查、JSON 校验和固定种子重放通过。
  • 完成至少 1000 次批量人生模拟并记录事件覆盖率、平均寿命、结局分布和异常。
  • 在 Linear RR-95 记录 Commit SHA、文件、测试、部署、移动端验证和下一审核闸门。

十九、与 WORLD 的复用映射

人物状态

迁移为怪物档案、属性、经历、关系、伤势、装备和长期标签。

事件条件

迁移为地点、行动、属性、资源、关系、世界状态和时间条件。

Effect

迁移为统一结构化 Effect，由 WORLD Engine 执行。

链式事件

迁移为狩猎、探索、招募、领地和世界异步事件链。

人生档案

迁移为怪物持续车卡、世界经历、驻守史和可重放事件日志。

Simulation

迁移为 Event Composer 频率、重复度、经济和成长模型的批量验证。
