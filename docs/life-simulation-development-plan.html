M00 · Life Simulation Engine · Plan v3

现实人生模拟引擎开发计划 v3

本项目先以现实世界单页单机人生重开游戏验证玩法，再沉淀为 WORLD 可复用的角色、事件、关系、档案与 Simulation 引擎。无数据库，内容使用 JSON，状态保存在浏览器本地。

一、验证目标

  • 人生按年、恋爱按周的双节奏是否好玩。
  • 事件、Command、延迟后果、关系与结局能否完全数据驱动。
  • 档案、成就和多周目是否形成持续游玩动力。
  • 相同种子是否可重放，批量人生是否可统计和平衡。
  • Engine 是否可迁移到 WORLD NPC 与世界事件系统。

二、核心循环

创建人物 → 年度事件 → 选择 → Command 原子结算 → +52周
                  ↓
             进入恋爱阶段
                  ↓
恋爱周事件 → 选择 → 关系维度变化 → +1周 → 关系定性
                  ↓
结局 → 九维评分 → 成就 → 档案 → 重开

三、系统架构

UI / SVG Renderer
       ↓
Life Controller
       ↓
Event Selector / Command Executor / Relationship Engine
       ↓
Canonical Clock / Metric Engine / Ending Engine / Archive
       ↓
JSON Content + localStorage v3
       ↓
Simulation Runner

四、人物模块

LifeState 拆分为 Identity、Body、Mind、Career、Finance、Relationships、Health、Inventory、Location、History。M00 只实现必要字段，但禁止重新退化为不可维护的扁平对象。

五、时间与中断

  • totalWeeks 是唯一权威时间。
  • 年度选择 +52 周；恋爱选择 +1 周。
  • 危机为临时插入，不改变当前 stage；按危机事件自己的 timeScale 推进后继续原阶段。
  • 冷却与 scheduled 全部存绝对总周数。

六、关系玩法

八项关系维度：attraction、love、trust、conflict、dependence、respect、passion、commitment。状态机为 potential、dating、exclusive、cohabiting、engaged、married，以及 paused、broken_up、no_contact。

  • 单段目标 6–16 次选择。
  • 软上限 26 周，硬上限 52 周。
  • 空白周自动快进。
  • 状态转换由关系维度、前置状态和明确承诺事件共同判定。

七、事件内容

事件由 Scene、Narration、Choice、Commands、Future Commands、Probability Modifiers 和 AI Tags 构成。

  • 年度人生事件：60–80。
  • 恋爱周事件：36–48。
  • 恋爱链式节点：12–18。
  • 危机中断事件：8–12。
  • 结局前事件：6–8。
  • 首轮总事件不少于 120。

八、结局与评分

Ending 只读取九个高层指标：achievement、relationship、health、money、legacy、freedom、knowledge、contribution、adventure。完整公式由 data/life-derived-metrics.json 注册，未知指标直接失败。

九、开发阶段

  1. P0：冻结 Contract v3、Event/Save Schema v3、Metric Registry v3、Ending Rules v3。
  2. P1：测试优先实现 canonical clock、Command Executor、运行时校验和 v2→v3 迁移。
  3. P2：实现人物模块、关系八维与 Relationship Engine。
  4. P3：迁移年度事件，完成年度垂直切片。
  5. P4：实现恋爱周阶段、危机插入和关系定性。
  6. P5：实现九维 Metric 与 Ending Engine。
  7. P6：实现本地存档、导入导出、档案与成就。
  8. P7：实现 runLife 与 runBatch，执行至少 10,000 次批量模拟。
  9. P8：移动端 UI、程序化 SVG 和三轮实际游玩审核。

十、测试矩阵

  • 年度 +52 周、恋爱 +1 周、51→52 跨年。
  • 危机插入不改变 stage，按自身 timeScale 推进。
  • 冷却与 scheduled 截止周准确。
  • 所有 Command 参数严格校验，非法操作整次回滚。
  • 关系维度截断、状态转换和硬上限定性。
  • Metric 公式可确定性计算，未知指标报错。
  • Ending 只读取高层 Metric。
  • v2→v3 存档迁移不丢档。
  • 相同 seed、policy、content 得到相同结果。

十一、质量门槛

  • 单次人生 8–15 分钟。
  • 同事件单生重复不超过 2 次；连续同类不超过 3 次。
  • 伪选择率低于 15%。
  • 恋爱卡死率为 0。
  • 10,000 次 Simulation 中无不可解释异常；单一主结局原则上不超过 35%。

十二、完成标准

Engine、单页 Web、JSON 内容、SVG、自动测试、10,000 人生 Simulation、本地存档、档案、成就、移动端审核和 Linear 同步全部完成；实现与 Contract v3、Schemas v3、Metric Registry v3、Ending Rules v3 一致。
