怪物卡属性、成长资质、插画与交易模型

怪物是一张可长期成长、可生成实体快照、可由玩家交易的数字角色卡。怪物只有名字与种族作为固定身份，没有职业；技能全部由后天行动、事件和训练习得。

一、固定身份

每只怪物永久保存唯一怪物 ID、名字、种族、原始生成种子、出生时间、初始拥有者、当前拥有者和交易历史。玩家可见的核心固定身份只有：

名字
种族

名字在确认后锁定；种族不可更改。怪物不得拥有职业字段、职业等级或职业限定技能树。

二、显式基础属性

实体卡和数字卡统一只显示四项基础属性：

力量 Power

影响攻击强度、破坏、负重和追击中的压制能力。

体魄 Vitality

影响生命、承伤、恢复、坚守和重伤抵抗。

敏捷 Agility

影响先手、命中、闪避、突袭、撤退和追击。

感知 Instinct

影响侦查、识破伏击、战术判断和技能触发修正。

四项属性必须全部进入战斗结算。不得加入只用于展示、却不影响结算的显式属性。

强攻：力量 + 体魄
突袭：敏捷 + 感知
坚守：体魄 + 感知
追击：敏捷 + 力量

具体权重、装备修正和地点修正由后续战斗 Simulation 验证，不在本规范中冻结。

三、等级与技能

等级代表成长阶段，不直接作为战斗加值。MVP 建议先开放 1–10 级，长期上限可扩展到 20 级。

技能全部后天习得，来源包括行动重复、事件选择、导师、装备使用、地点训练、其他怪物传授、技能书和重大经历。

已学技能库：MVP 最多 8 个
当前装备技能：最多 3 个
实体卡显示：最多 3 个当前技能

同一种族可以形成完全不同的技能组合，但其身份始终只是该种族个体，不产生“战士、猎人、法师”等职业。

四、隐藏成长资质

每只新怪物出生时生成四项固定隐藏成长资质：

力量成长
体魄成长
敏捷成长
感知成长

内部值建议使用 1–100。资质在出生时固定，不能训练、洗点、交易后重置或通过客户端修改。

当前显式属性
= 种族基础值
+ 等级成长结果
+ 后天训练
+ 永久事件修正

等级成长结果受对应隐藏成长资质影响。低等级时普通个体与极品个体差距较小；养成后差距逐渐通过显式属性显现，从而形成“极品哥布林”等稀有个体。

成长资质不能完全决定角色价值。设计目标是：种族基础、天生成长和玩家培养共同塑造最终能力。具体占比由 Simulation 决定。

五、稀有度与鉴定

稀有度表示天生成长潜力，不表示当前战斗力，也不自动赋予技能。

稀有度 = 四项隐藏成长资质的综合评级
技能强度 = 玩家后天培养、经历与搭配

出生时不公开精确成长值，也不直接显示稀有度。玩家通过养成逐渐获得鉴定信息：

等级 1–3：资质未知
等级 4–6：出现初步成长倾向
等级 7–9：形成较明确区间评级
等级 10：完成正式鉴定

即使完成鉴定，普通客户端也只显示区间或文字评级，例如“普通、优秀、卓越”，不公开精确内部值。成长总值区间和普通、优良、稀有、史诗、极品的概率分布属于后续 Simulation 项目，不在本规范中正式冻结。

六、隐藏外观基因

隐藏外观基因与成长资质完全分离。外观基因不参与战斗，只用于稳定生成同一只怪物的插画。

固定外观基因

基础体型、身高比例、头部比例、耳型、肤色、眼色、角、牙齿、毛发和基础斑纹。

经历外观

疤痕、伤势、装备、服饰、奖章、年龄感、姿态、表情和驻守地点背景。

固定外观基因创建后永久不变；经历外观通过事件追加。插画生成服务读取外观数据，但不需要读取隐藏成长资质。

七、实体怪物卡

实体卡正面只承载可快速识别和战斗所需的信息：

名字
种族
插画
等级
力量 / 体魄 / 敏捷 / 感知
最多 3 个当前技能
成长鉴定评级
怪物唯一 ID / QR

完整经历、关系、全部技能、精确成长值、交易历史和外观基因保存在数字档案中。

实体卡是数字怪物某一时点的版本快照，而不是永远同步的实时卡。每个快照记录怪物 ID、快照版本、打印时等级、属性、技能、打印日期和状态哈希。旧快照永久有效，可形成幼年卡、成长卡、成熟卡或重大事件纪念卡。

八、怪物交易

玩家交易的是怪物数字所有权及其完整成长记录，而不是单独交换一张图片。

随怪物转移：

  • 当前怪物数据与控制权；
  • 显式属性、技能、经历、伤势和成长鉴定；
  • 装备归属按正式交易规则处理；
  • 实体卡快照历史和交易历史。

不随怪物转移：原账号资源、领地、账号级成就和其他怪物。

只有空闲或休养状态的怪物可交易。当前主控、驻守、执行任务、被俘、受困或参与未结算冲突的怪物不得交易。创建者与历任拥有者记录永久保留，但公开展示应使用玩家公开名或匿名编号。

九、客户端与服务端边界

  • 精确成长资质只能保存在服务端，不发送给普通客户端；
  • 客户端只接收当前显式属性和已解锁的鉴定结果；
  • 升级、鉴定、交易和属性成长必须由服务端权威结算；
  • 外观基因可以由受控插画服务读取，但不得影响战斗；
  • 实体卡生成只读取公开卡面数据和快照元数据。

十、MVP 锁定项

固定身份：名字、种族
职业：无
显式属性：力量、体魄、敏捷、感知
技能：全部后天习得
已学技能：最多 8 个
当前装备技能：最多 3 个
成长资质：4 项隐藏固定值
鉴定：随等级逐步揭示，等级 10 完成
插画：独立隐藏外观基因
交易：转移数字怪物所有权
实体卡：版本快照

本规范锁定结构和产品边界。正式属性初值、成长公式、稀有度概率、技能数值、战斗权重、交易费用与市场限制仍需后续 Engine、Simulation 和审核。
