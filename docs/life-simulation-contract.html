RR-95 · Contract Freeze v2.1

人生模拟数据与规则契约 v2.1

本契约解决混合时间、恋爱状态机、阶段长度、派生指标与中断优先级。后续 Engine、内容、UI、测试、Simulation 与存档迁移必须以本页和对应 Schema 为准。

一、Canonical Clock

  • clock.totalWeeks 是唯一权威时间。
  • ageYears = floor(totalWeeks / 52)，weekOfYear = totalWeeks % 52，两者为派生值。
  • 年度选择成功后 totalWeeks += 52；周制选择成功后 totalWeeks += 1。
  • 进入或退出阶段不推进时间，不修改 totalWeeks，也不清零 weekOfYear。
  • 所有年龄条件读取派生 ageYears；所有计划事件以 dueAtTotalWeeks 判断。

二、阶段与关系状态机

stage: life | romance
romanceStatus:
none → potential → dating → exclusive → cohabiting → engaged → married
                    ↘ paused → broken_up → no_contact
  • 进入 romance：当前存在 role=partner_candidate 的关系，状态为 potential 或 dating，并由明确选择执行 enter_stage。
  • 退出 romance：married、broken_up、no_contact、对象移除、主动结束或硬上限定性。
  • 婚后日常、育儿与长期家庭事件属于 life/year；婚姻危机可创建新的 romance/week 子阶段。
  • 单段目标 6–16 个选择，软上限 26 周，硬上限 52 周。到达硬上限必须抽取最高优先级 relationship_resolution 事件。
  • 无主要事件的周由 Engine 自动快进到下一个有效周或到期 scheduled，不写入“平静周”玩家选择。

三、状态结构

LifeState
├─ id / seed / rngCursor / createdAt
├─ identity
├─ clock: totalWeeks / stage / stageStartedAtWeeks
├─ alive
├─ stats / resources
├─ relationships[]
├─ tags[] / flags{}
├─ experiences[] / timeline[]
├─ cooldowns{}
├─ scheduled[]
├─ derivedMetrics
├─ pendingEvent
└─ ending

ageYears、weekOfYear、stageWeek、净资产等均为派生值。事件不得直接写 derivedMetrics。

四、事件与中断优先级

  1. 结算到期 scheduled effects。
  2. 检查死亡与不可继续状态。
  3. 检查强制链式事件。
  4. 检查允许中断阶段的健康、法律和公共危机。
  5. 检查当前阶段里程碑与硬上限定性。
  6. 筛选 stage、timeScale、年龄、stageWeek、关系、标签、Flag、属性和资源。
  7. 排除 oncePerLife 与冷却事件。
  8. 最高 priority 层内按 weight 抽取。

只有标记 canInterruptStage=true 的非恋爱事件可以进入 romance 事件池。

五、Effect 与原子结算

白名单操作包括数值、资源、标签、Flag、身份、职业、教育、关系、经历、计划事件、enter_stage 与 exit_stage。选择结算顺序：

验证 choiceId → 深拷贝 → 即时 effects → futureEffects/nextEvent
→ cooldown/oncePerLife → experience/timeline → 按结算前 timeScale 推进时间
→ 重新计算派生指标 → Schema 校验 → 结束检查 → 原子保存

阶段切换 Effect 在本次选择中生效，但本次推进单位取事件声明的 timeScale，避免 enter_stage 后同一次选择误按周推进。

六、派生指标定义

  • businessSuccess：创业相关经历分、企业资产与持续盈利归一化到 0–100。
  • peopleHelped：带有 helped_person 的唯一经历数量。
  • familyStability：存续家庭关系的亲密与信任均值，扣除冲突与断裂惩罚。
  • partnerCloseness：当前或最后一段长期伴侣关系 closeness。
  • relationshipYears：长期伴侣关系累计周数 ÷ 52。
  • lifeVariety：经历类型、职业、地区和重大活动的去重覆盖率。
  • lateLifeGrowth：50 岁后属性、教育、事业和关系改善的归一化增量。
  • turningPoints：标记 turning_point 的唯一经历数量。
  • unresolvedGoals：active_goal 且未有 completed_goal 对应项的数量。
  • loneliness：100 减去有效亲密关系支持度，截断到 0–100。
  • careerChanges / educationEvents / riskTurns / recoveryTurns：对应经历标签计数。
  • peakWealth / largestLoss：时间线资产快照的历史峰值与最大峰谷跌幅归一化值。

七、冷却与计划事件

冷却保存为 {remaining, unit}，运行时统一换算为 cooldownUntilTotalWeeks。scheduled 保存 dueAtTotalWeeks。年单位按 52 周精确换算。

八、存档

localStorage Key 为 realmruckus.world.life.v2，schemaVersion=2。存档保存 totalWeeks、stage、stageStartedAtWeeks、关系状态、seed、rngCursor、scheduled 与基础历史；派生值允许保存缓存，但加载后必须重新计算并覆盖。

九、质量与测试闸门

  • 51→52 周跨年、任意 weekOfYear 退出恋爱、年度推进保持周偏移。
  • 硬上限定性、空白周快进、危机中断和恢复阶段。
  • 年/周冷却、跨阶段 scheduled、死亡优先级。
  • v1→v2 迁移、保存恢复、固定种子重放。
  • 恋爱卡死率 0；伪选择率低于 15%；单一结局 Simulation 占比原则上不超过 35%。

十、冻结状态

Contract v2.1 已冻结。删除字段、改变 canonical clock、阶段状态机、结算顺序或存档 Key 均属于破坏性变更。
