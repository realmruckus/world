Development reference

开发难度与路线

当前 WORLD 已从传统 MMORPG 收敛为异步共享世界、地点占领、一次性结果结算和长期怪物成长的网页卡牌 RPG。综合难度：中等偏高。

系统难度主要风险
地点与容量低至中不同规模地点是否都有价值。
怪物、驻军与装备中状态互斥、重复派驻和角色保护。
World Tick 与事件投影中等偏高可见性、排序、重复结算和一致性。
一次性抢地中胜率、损失、解释性和单一最优策略。
经济 Simulation高通胀、滚雪球、维护、新玩家追赶。
事件内容高重复感、选择质量和奖励惊喜。

Simulation 必须验证

  • 金币和资源是否持续膨胀。
  • 大地点是否必然碾压小地点。
  • 驻军维护是否形成真实取舍。
  • 进攻是否值得冒险但不会成为无限套利。
  • 后加入玩家能否追赶。
  • 地点易主率、平均驻军率和玩家破产率是否合理。

原型顺序

1. World Tick 与追加式事件模型
2. 地点、怪物和驻军数据结构
3. 结构化 Effect 与一次性领地结算
4. 经济与扩张 Simulation
5. Event Composer 与奖励保底
6. 本地可玩垂直切片
7. 账号、云存档和服务端权威结算
8. 异步事件投影
9. 内容扩充与公开测试

在 Simulation 稳定前，不制作完整美术、不开放自由交易、公会战争或复杂 PvP。
