事件检定、战斗随机与怪物迭代边界

WORLD 的盲盒感来自隐藏成长资质、事件结果和有限战况波动。随机性制造悬念，但不能覆盖长期养成、阵容质量、策略和地点优势。

一、事件围绕四项属性

每个事件使用一项主要属性、最多一项辅助属性，并结合后天技能、装备、经历、特性和有界随机。

事件检定值
= 主要属性
+ 辅助属性 × 0.5
+ 技能修正
+ 装备修正
+ 经历／特性修正
+ 有界随机

力量用于破坏、搬运与压制；体魄用于承受、恢复与坚持；敏捷用于追击、闪避和快速行动；感知用于侦查、识破和判断。事件不得同时无差别检查全部四项属性。

事件至少支持大成功、成功、代价成功和失败。失败仍应提供训练进度、经历、线索或保底积累，避免一次选择完全没有成长。

二、战斗随机原则

战斗保留有限、可解释、可重放的随机性。一般战况波动目标约为 0.90–1.10，正式范围由 Simulation 决定。

  • 养成与阵容决定主要结果；
  • 地点、策略、侦查和技能协同决定可控修正；
  • 随机只改变临界区间，不应轻易覆盖超过约25%的硬实力差距；
  • 服务端保存公式版本、完整输入、防守快照和随机种子；
  • 战后向玩家解释先手、坚守、侦查、地点和战况波动等主要原因。

感知可以压缩负面波动范围，使高感知队伍更稳定，而不是简单提供另一项固定攻击力。

三、四项属性进入战斗

攻击 = 力量 + 0.35 × 敏捷
防御 = 体魄 + 0.35 × 感知
先手 = 0.70 × 敏捷 + 0.30 × 感知
战术 = 0.70 × 感知 + 0.30 × 敏捷

以上是待验证的初始结构，不冻结正式系数。策略通过改变四项属性的权重形成差异：

强攻

力量40%、体魄25%、敏捷20%、感知15%。

突袭

力量20%、体魄10%、敏捷35%、感知35%。

坚守

力量15%、体魄40%、敏捷10%、感知35%。

追击

力量30%、体魄10%、敏捷40%、感知20%。

四、多怪物战斗

多人战斗不得直接比较全部属性总和。建议使用：

队伍评分
= 队伍平均个体评分
× 人数递减系数
× 前线宽度效率
× 地点修正
× 策略修正
× 技能协同
× 战况随机

人数递减系数的初始候选为 人数^0.85。因此8只怪物相对5只怪物仍有明显人数优势，但不会机械地获得60%的完整线性优势。

地点同时记录驻军容量和前线宽度。多余驻军不一定完整进入当前交锋，但可以提供轮换、伤害分摊、持久战和防止一次性清空的价值。

设计目标是：5只培养良好、侦查成功且策略匹配的怪物可以挑战8只普通驻军；双方接近时结果可能为小胜、僵持或小败，但人数优势不得失去意义。

五、繁殖与怪物迭代

MVP 不开放繁殖。公开 MVP 稳定后，才评估繁殖、谱系和血统收藏。

后代成长资质
= 父母对应资质的加权均值
+ 向同族平均值回归
+ 小幅随机变异

繁殖不保证后代强于父母。优秀父母提高产生优秀后代的概率，普通父母仍保留极低概率产生极品个体。四项成长资质分别遗传，不能只继承总稀有度。

固定外观基因可以组合和突变；疤痕、装备、奖章等经历外观不得遗传。后代拥有新的怪物 ID、名字、成长记录和交易历史。

六、不采用吞噬式合成

怪物有名字、经历、实体快照和交易历史，因此不采用“消耗多只怪物升级一只怪物”的传统合成。后续可研究：

  • 繁殖与谱系；
  • 技能传承；
  • 导师训练；
  • 血统研究；
  • 提高训练效率的培育资源。

传承和训练不得直接转移或重置隐藏成长资质。

七、MVP锁定与待验证项

事件：1项主要属性 + 最多1项辅助属性
事件结果：大成功 / 成功 / 代价成功 / 失败
战斗：有限随机、可解释、可重放
多人结算：平均质量 + 递减人数 + 前线宽度 + 地点 + 策略 + 协同
繁殖：MVP不开放
吞噬式合成：不采用

正式事件难度、属性系数、人数指数、前线宽度效率、随机范围、繁殖概率、回归均值比例和突变分布，必须由 M10 领地结算实验室与 M13 经济 Simulation 验证后另行批准。
