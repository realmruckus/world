Character creation

持续车怪物卡

怪物卡不是开局填写完成的角色表，而是玩家通过长期选择不断生成的角色记录。

初始框架

创建时只确定种族、基础属性倾向、天生能力、弱点、初始装备和少量生存选择。发展方向由实际事件逐渐形成，而不是复杂点数分配。

怪物卡组成

层级内容
固有数据种族、体型、基础属性、天生能力和弱点。
成长数据等级、经验、生命、行动力、狩猎、侦查、探索、制作、交涉等。
经历标签逃离狼群、第一次战败、占领山洞、救下怪物、驻守地点等。
性格倾向谨慎、鲁莽、善意、贪婪、求生者、收藏癖等由重复选择形成。
关系与世界记录同伴、群落、领地、仇敌、声望、被俘和营救记录。

每次选择的统一输出

1. 即时奖励或损失
2. 属性或技能变化
3. 经历记录
4. 性格倾向变化
5. 关系变化
6. 地点或世界状态变化

文本与规则分离

事件文本负责表达，结构化 Effect 负责实际修改。文本层不得自行创造未批准的属性、技能、物品、怪物和数值。

驻军继续成长

驻守怪物保留完整个体历史。地点防守、失守、被俘、逃回和营救都可以形成经历与性格倾向，因此驻军不是无名战力数字。

招募怪物

招募角色可以带预设背景、经历或倾向加入，但加入后使用同一成长系统。新建怪物更接近空白角色，招募怪物则从已有故事继续塑造。

MVP 范围

  • 3 个基础种族。
  • 每只怪物 4–6 项核心属性或技能。
  • 6–10 个可形成特性。
  • 20–30 个经历标签。
  • 每个选择最多修改 1–3 个长期字段。
  • 不做复杂心理疾病、繁殖、血统或永久人格锁死。

核心原则：每个玩家选择至少改变当前状态、长期成长或世界关系之一；重要选择必须在怪物卡上留下可追踪痕迹。
