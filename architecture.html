Architecture

整体架构

仓库身份

realmruckus/world 是 WORLD 独立游戏的唯一实现与产品网站来源。根目录即网站根目录，不使用额外 website 子目录。

依赖边界

WORLD 消费 Core 的通用 Mechanic、Effect、产品元数据与长期基线，通过 dependencies.lock.json 固定明确 Commit，不长期跟踪 main。

运行分层

HTML 页面与卡牌式 UI
        ↓
js/app.js 交互适配
        ↓
js/engine.js
├─ World Tick
├─ 确定性随机
├─ 怪物角色与持续车卡
├─ 结构化 Effect
├─ Event Composer
├─ 地点、驻军与领地结算
└─ 经济计算
        ↓
未来服务端权威事务、数据库与审计日志

网站内容

  • index.html：产品入口与核心定位
  • play.html：可运行垂直切片
  • architecture.html：技术与仓库边界
  • docs/：所有设计文档网页
  • releases.html：版本和迭代记录
  • status.html：长期产品状态摘要；动态任务仍以 Linear 为准

权威结算原则

浏览器原型仅验证体验。正式账号、奖励、招募、领地易主与经济变化必须由服务端权威结算，并使用幂等行动 ID、版本化模板、随机种子和追加式事件日志支持重放与审计。
