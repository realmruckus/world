Modular development roadmap

WORLD MVP 模块化开发计划

每个模块必须同时交付 Engine 能力、独立 Web 验证页、自动测试和版本记录。依赖模块未通过网页验证与测试前，不进入下一模块。

统一完成标准

  • Engine

    规则、状态和结算逻辑不得写死在页面中。

  • Web 验证

    每个模块提供独立页面，可操作、可检查输入与输出。

  • Tests

    覆盖确定性、边界、非法状态、重复请求和数据一致性。

  • Release

    更新版本记录，并在 Linear 写入 Commit、测试和验证结果。

Milestone 1：Playable Character Core

M01 调试台与状态检查器

/dev.html：Engine 版本、World Tick、JSON 状态、事件日志、随机种子、重置、导入和导出。

Linear：RR-70

M02 账号、怪物档案与角色切换

/monsters.html：账号内多怪物、创建、命名、种族、主控切换和状态互斥。

Linear：RR-71

M03 怪物卡与持续车卡

/monster-card.html：属性、技能、特性、经历、关系、装备、伤势、驻守史和结构化 Effect。

Linear：RR-72

Milestone 2：Event-Driven World

M04 Event Composer

/events.html：条件、权重、冷却、稀有度、保底、模板版本、频率和重复检查。

Linear：RR-73

M05 狩猎垂直切片

/hunt.html：基础收益、稀有事件、装备、招募、成长、保底和地点差异。

Linear：RR-74

M06 探索、采集与休养

/actions.html：新地点、遗迹、资源、材料、恢复和营地事件。

Linear：RR-75

M07 世界地图与地点

/world.html：区域、地点、发现状态、容量、资源、防御修正和事件池。

Linear：RR-76

Milestone 3：Territory and Garrison

M08 驻军配置

/garrison.html：派驻、撤回、容量、防守快照、适应性、维护和防守策略。

Linear：RR-77

M09 招募与新怪物养成

/recruit.html：养成与招募两种来源、背景经历、栏位和经济成本。

Linear：RR-78

M10 领地结算实验室

/territory-lab.html：五档结果、攻防快照、地点加成、侦查、策略、批量运行和固定种子重放。

Milestone 4：Asynchronous Multiplayer

M11 World Tick 与异步投影

/timeline-lab.html：玩家 A/B 双时间线、事件可见性、临时占领、汇合冲突和历史收益不回滚。

Linear：RR-80

M12 多玩家权威后端

/multiplayer-lab.html：账号、快照、事件、投影游标、幂等请求、审计日志和重新登录恢复。

Linear：RR-81

M13 经济与扩张 Simulation

/simulation.html：多玩家、多 Tick、收益、维护、进攻成本、追赶、统计和参数对比。

Linear：RR-82

Milestone 5：Public MVP

M14 内容扩充工具

/content-tools.html：事件编辑、条件预览、Effect、奖励池、覆盖率、重复度和无效 ID 检查。

Linear：RR-83

M15 MVP 用户体验整合

/play.html：创建怪物、事件成长、装备、招募、占领、驻军和异步冲突的完整流程。

Linear：RR-84

M16 部署、监控与公开测试

GitHub Actions、测试、链接和 Schema 检查、部署、错误上报、版本、存档迁移和反馈入口。

Linear：RR-85

开发顺序

M01 → M02 → M03 → M04 → M05 → M06 → M07 → M08
→ M09 → M10 → M11 → M12 → M13 → M14 → M15 → M16
